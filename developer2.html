The Problem CI/CD Solves

Without it vs. with it

❌ Without CI/CD
✗Merge once a week — huge conflicts to untangle
✗Manual testing before every release
✗Deployment is a stressful team event
✗Bugs found days or weeks after they were introduced
✅ With CI/CD
✓Merge small changes daily — conflicts are tiny
✓Automated tests run on every commit
✓Deployment is automatic and routine
✓Bugs caught within minutes of being pushed
Real-World Impact

Why Agile teams adopt it

↑ 46× more frequent deployments than teams without CI/CD
↓ 440× faster lead time from commit to production
↓ 5× lower change failure rate with automated testing
This Page's Git Journey

How Student 2 got here

🍴

Fork the repo

Student 2 forks Student 1's repository, creating an independent copy under their own GitHub account.

🌿

Create a feature branch

Branches off main with git checkout -b feature/student2-page to keep work isolated.

💾

Commit student2.html

Adds this file and pushes: git add student2.html && git commit -m "feat: add student2 page"

🔁

Open a Pull Request

PR targets Student 1's main branch. Student 1 reviews, approves, and merges.

🚀

GitHub Pages redeploys

Merge triggers automatic redeployment. This page is now live — that's CI/CD in action.
