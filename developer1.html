Core Concepts

The three pillars

🔀

Continuous Integration

Developers merge code to a shared trunk frequently — often daily — so problems are caught early and conflicts stay small.

📦

Continuous Delivery

Every merged change is automatically built and deployed to a staging environment, ready to release with one click.

🚀

Continuous Deployment

Staging tests pass? The code ships to production automatically — no manual release step, no waiting.

Trunk-Based Development

How this demo was built

1

Student 1 creates the repo

Sets up index.html, student1.html, enables GitHub Pages on the main branch.

2

Student 2 forks & branches

Forks the repo, creates a feature/student2-page branch, adds student2.html.

3

Pull Request & Review

Student 2 opens a PR targeting the original repo. Student 1 reviews and merges.

4

GitHub Pages redeploys

The merge to main automatically triggers a new deployment — both pages go live together.
